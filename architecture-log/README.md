# Architecture Log

This folder is maintained by the **Senior Coder Agent**. It contains:

## Files

### `current-architecture.md`
Living document describing the current system architecture. Updated after every project cycle. The Senior Coder reads this at the start of every engagement.

### Issue Logs (per feature)
Named: `{number}-{feature}-issues.md` (e.g., `001-lane-config-issues.md`)

Each entry includes:
- **Issue** — what was found
- **Phase** — spot-check, final review, or reviewer feedback
- **Severity** — Critical / High / Medium / Low
- **Root Cause** — why it happened
- **Resolution** — how it was fixed
- **Prevention** — how to avoid this in future projects

### Decision Records
Named: `decision-{number}-{short-description}.md`

Each entry includes:
- **Context** — what prompted the decision
- **Options Considered** — alternatives evaluated
- **Decision** — what was chosen and why
- **Consequences** — tradeoffs accepted
